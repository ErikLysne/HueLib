    Detailed Description

    The HueBridge class represents the physical bridge connected to the network.

    The Hue bridge is the link between Hue lights and the devices and applications that interface with them. To access the bridge, an application requires a valid username generated from the bridge. A username can be supplied with the constructor, or one can be generated by calling the link() function.

    The minimum requirement to create a HueBridge object is the IP address of the bridge. In this case, the HueBridge object will be constructed without a valid username, and one must be created using the link() function. When link() is called, the user has 10 seconds to press the link button on the bridge. When the button is pressed, a new username will be generated and stored in a file in the build folder. link always checks if a valid username has already been created before trying to create a new one, so pressing the button must only be done the first time a new application is running.

    HueBridge* bridge = new HueBridge("10.0.1.14");
bridge->link();

    If you want to supply an existing username instead of creating a new one, this can be added as a second argument.

    HueBridge* bridge = new HueBridge("10.0.1.14", "1028d66426293e821ecfd9ef1a0731df");

    HueBridge needs access to QNetworkAccessManager. By default, it will create one internally, but if this resource is required elsewhere, it can be created externally and a reference to HueBridge can be provided. Use a third argument to provide a pointer to a QNetworkAccessManager.

    QNetworkAccessManager* nam = new QNetworkAccessManager();
HueBridge* bridge = new HueBridge("10.0.1.14", "1028d66426293e821ecfd9ef1a0731df", nam);

    testConnection() can be used to test the connection to the bridge.

    HueBridge* bridge = new HueBridge("10.0.1.14");
bridge->link();

HueBridge::ConnectionStatus status;
bridge->testConnection(status);

if (status == HueBridge::Success)
    qDebug() << "Connected to bridge.";

    See also link() and testConnection(ConnectionStatus& status).

    Member Type Documentation

    enum HueBridge::ConnectionStatus

    This enum defines the status that can be returned by testConnection(ConnectionStatus& status). Test for various cases to provide the user with a hint about possible reason if testConnection(ConnectionStatus& status) did not give HueBridge::Success.

    ConstantValueDescription
    HueBridge::Success0Sent a command and received expected reply. Connection successful.
    HueBridge::TimedOut1Tried to send a command, but did not receive a reply within the specified timeout. Most likely caused by incorrect ip address. Alternatively, timeout period can be extended by setNetworkRequestTimeout()
    HueBridge::JsonError2Received a reply containing a JSON error. Most likely caused by an incorrect username. Use getLastError() to check the error.
    HueBridge::HttpError3Received a reply with HTTP status different from 200. Note: replies with 200 status can still contain JSON errors.
    HueBridge::Unknown4An unkown error has occured. Should normally never happen.

    See also testConnection(ConnectionStatus& status) and testConnection().

    QString HueBridge::link(QString appName = "C++ HueLib app", QString deviceName = "")

    The first time link() is called in an application, the user will be prompted to press the link button on the Hue bridge. The user then has 10 seconds to press the link button, after which the function will time out and return an empty QString.

    If the user pressed the link button before the timeout, a new username is created and returned. Additionally, a hidden folder .huelib will be created in the build directory with a username.txt file containing the newly created username. This also sets the username on the bridge.

    link() always checks if a username.txt file exists in the .huelib folder before attempting to create a new user, so calling link() immediately after constructing HueBridge means the username never has to be set explicitly.

    appName and deviceName sets a meaningful name for the created username. Names are stored as "app_name#device_name".

    Note: since the security update in Hue API 1.31 released April 9. 2019, the list of registered usernames is no longer accessible from the whitelist stored on the bridge. Instead, users must go to https://account.meethue.com/apps and register their bridge to see the list of usernames.

    HueBridge* bridge = new HueBridge("10.0.1.14")
bridge->link()

    Returns an existing username if one is found, or the newly created username if one was created. Otherwise returns an empty QString.

    void HueBridge::setBridgeCommandBlockTime(const int milliseconds)

    Sets the time (in milliseconds) the bridge is blocked for after sendRequest() has been called without a specified object, such as when calling discover functions. This is set to limit traffic to the throughput of the bridge. Setting a low value can lead to higher throughput, but can lead to more frequent timeouts and potentially make the bridge less responsive.

    Block time is specified by milliseconds.

    See also setLightCommandBlockTime(), setGroupCommandBlockTime(), and setNetworkRequestTimeout().

    void HueBridge::setGroupCommandBlockTime(const int milliseconds)

    void HueBridge::setNetworkRequestTimeout(const int milliseconds)

    Sets the time (in milliseconds) until timeout after calling sendRequest().

    Note: Reducing the timeout period for network requests will not make the bridge more responsive.

    Timeout period is specified by milliseconds.

    See also setLightCommandBlockTime(), setGroupCommandBlockTime(), and setBridgeCommandBlockTime().
